## 🚀 第一步：修改配置（云托管适配）

### 1.1 修改 `config/config.go`

让后端支持环境变量 `PORT`（云托管默认分配 80 端口）：

```go
package config

import (
    "os"
    "time"
)

type Config struct {
    ServerPort string
    DBPath     string
}

func Load() *Config {
    port := os.Getenv("PORT")
    if port == "" {
        port = ":8080"
    } else {
        port = ":" + port
    }
    dbPath := os.Getenv("DB_PATH")
    if dbPath == "" {
        dbPath = "data/recipes.db"
    }
    return &Config{
        ServerPort: port,
        DBPath:     dbPath,
    }
}

// DailyRecommendExpiry defines how often daily recommendation rotates
const DailyRecommendExpiry = 24 * time.Hour
```

### 1.2 修改 `Dockerfile`

确保监听 80 端口：

```dockerfile
# Stage 1: Build
FROM golang:1.25-alpine AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 go build -o recipe-server .

# Stage 2: Run
FROM alpine:3.19
WORKDIR /app

# Runtime dependencies
RUN apk add --no-cache ca-certificates tzdata

# Copy binary
COPY --from=builder /app/recipe-server .

# Create necessary directories
RUN mkdir -p data uploads

# Copy seed images for initial image assignment
COPY --from=builder /app/images ./images

# Expose API port (80 for WeChat Cloud Hosting)
EXPOSE 80

# Use a non-root user for security
RUN adduser -D appuser && chown -R appuser:appuser /app
USER appuser

# Start with PORT=80 for cloud hosting
ENV PORT=80
CMD ["./recipe-server"]
```

## ☁️ 第二步：微信云托管部署后端

### 2.1 开通云托管

1. 打开 **微信开发者工具**
2. 用你的小程序 AppID（`wx5c394ff10c73e087`）打开 `wechat-recipe-app` 目录
3. 点击顶部工具栏 **「云开发」** → **「云托管」**
4. 首次使用按提示开通（按量计费，有免费额度）

### 2.2 创建服务

1. 在云托管面板点击 **「新建服务」**
2. 填写信息：
   | 字段 | 值 |
   |------|-----|
   | 服务名称 | `recipe-api` |
   | 部署方式 | **代码包上传** |
   | 代码包 | 打包 `e:\GO\recipe\wechat\` 目录为 `.zip` 上传 |
3. 点击 **「部署」**，等待 3-5 分钟构建完成

> 💡 **推荐使用 Git 仓库部署**：将项目推送到 GitHub/Gitee，在云托管绑定仓库，以后 `git push` 即自动部署。

### 2.3 上传代码包操作步骤

```bash

cd e:\GO\recipe\wechat

# 2. 打包（排除小程序前端和已编译文件）

#    - main.go
```

## 🔗 第四步：配置服务器域名白名单

1. 登录 [微信小程序管理后台](https://mp.weixin.qq.com)
2. 左侧菜单 **「开发」** → **「开发管理」** → **「服务器域名」**
3. 点击 **「修改」**：

   | 域名类型 | 添加的值 |
   |---------|----------|
   | `request合法域名` | `https://recipe-api-xxxxxxxxx.sh.run.tcloudbase.com` |
   | `uploadFile合法域名` | `https://recipe-api-xxxxxxxxx.sh.run.tcloudbase.com` |

4. 点击保存（每个月只能修改 5 次）

> ⚠️ **注意**：微信小程序要求所有 API 请求必须是 HTTPS，且域名已备案。云托管域名自动满足这两个条件。

## 📤 第五步：上传 & 发布小程序

### 5.1 在开发者工具中测试

1. 确保云托管服务已启动（正常看到日志输出）
2. 在微信开发者工具中逐个测试：
   - ✅ 首页每日推荐加载正常
   - ✅ 分类列表显示正常
   - ✅ 菜谱详情能打开
   - ✅ 收藏功能正常
   - ✅ 购物清单增删改正常
   - ✅ 图片上传正常

### 5.2 上传代码

1. 微信开发者工具 → 点击顶部工具栏 **「上传」** 按钮
2. 填写：
   - **版本号**：`1.0.0`
   - **项目备注**：`首次发布 - 包含每日推荐、分类浏览、收藏、购物清单功能`
3. 点击确认上传

### 5.3 提交审核

1. 登录 [微信小程序管理后台](https://mp.weixin.qq.com)
2. 左侧菜单 **「管理」** → **「版本管理」**
3. **「开发版本」** 区域找到刚上传的 `1.0.0` 版本
4. 点击 **「提交审核」**
5. 填写审核信息：
   - **功能配置**：勾选所有用到类目
   - **测试账号**：提供一个可登录的体验账号（如有）
   - **功能截图**：上传各页面截图
6. 提交，等待审核（通常 1-3 个工作日）

### 5.4 发布上线

审核通过后：
1. 回到 **「版本管理」** 页面
2. 在 **「审核版本」** 中找到通过审核的版本
3. 点击 **「发布」** → 确认

🎉 **发布成功！用户即可在微信搜索到你的小程序！**

## ⚠️ 上线前必须处理的问题

### 🔴 高优

| 问题 | 说明 | 修改位置 |
|------|------|---------|
| **用户 ID 写死** | `userId: 'demo_user_001'` 导致所有用户共享收藏和清单 | `app.js` 改用 `wx.login()` 获取 openid |
| **SQLite 数据丢失** | 容器重启数据会丢 | 云托管后台开通 MySQL 数据库，修改 `database.go` |

### 🟡 中优

| 问题 | 说明 |
|------|------|
| **分类封面图** | `AssignImages` 使用 Unsplash 外链图，建议换成云存储图片 |
| **API 没有鉴权** | 当前所有接口无用户身份校验，任何人都可操作 |

### 🟢 低优 / 可选

| 优化 | 说明 |
|------|------|
| 接入微信云数据库 | 替代 SQLite，数据不丢失 |
| 图片上传改用云存储 | 微信云存储 `wx.cloud.uploadFile` |
| 增加分类管理后台 | 方便运营维护菜谱数据 |
