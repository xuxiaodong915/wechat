### config/config.go — MySQL 配置（环境变量驱动）

```go
package config

import (
    "fmt"
    "os"
    "time"
)

type Config struct {
    ServerPort string
    DBHost     string
    DBPort     string
    DBUser     string
    DBPass     string
    DBName     string
}

func Load() *Config {
    port := os.Getenv("PORT")
    if port == "" {
        port = ":8080"
    } else {
        port = ":" + port
    }
    return &Config{
        ServerPort: port,
        DBHost:     getEnv("DB_HOST", "127.0.0.1"),
        DBPort:     getEnv("DB_PORT", "3306"),
        DBUser:     getEnv("DB_USER", "root"),
        DBPass:     getEnv("DB_PASS", ""),
        DBName:     getEnv("DB_NAME", "recipes"),
    }
}

// DSN returns MySQL connection string
func (c *Config) DSN() string {
    return fmt.Sprintf("%s:%s@tcp(%s:%s)/%s?charset=utf8mb4&parseTime=True&loc=Local",
        c.DBUser, c.DBPass, c.DBHost, c.DBPort, c.DBName)
}

func getEnv(key, fallback string) string {
    if v := os.Getenv(key); v != "" {
        return v
    }
    return fallback
}

// DailyRecommendExpiry defines how often daily recommendation rotates
const DailyRecommendExpiry = 24 * time.Hour
```

### database/database.go — MySQL 驱动

```go
package database

import (
    "go-chain/config"
    "go-chain/models"

    "gorm.io/driver/mysql"
    "gorm.io/gorm"
    "gorm.io/gorm/logger"
)

var DB *gorm.DB

func Init(cfg *config.Config) error {
    var err error
    DB, err = gorm.Open(mysql.Open(cfg.DSN()), &gorm.Config{
        Logger: logger.Default.LogMode(logger.Warn),
    })
    if err != nil {
        return err
    }

    // Auto-migrate tables (first run creates all tables automatically)
    err = DB.AutoMigrate(
        &models.Category{},
        &models.Recipe{},
        &models.Favorite{},
        &models.ShoppingItem{},
    )
    if err != nil {
        return err
    }

    return nil
}

func GetDB() *gorm.DB {
    return DB
}
```

1. 打开 **微信开发者工具**
2. 用小程序 AppID `wx5c394ff10c73e087` 打开 `wechat-recipe-app` 目录

6. 记下 MySQL 的连接信息（内网地址、端口、用户名、初始密码）

### 1.2 创建后端服务

1. 在云托管面板点击 **「新建服务」**
2. 填写：

   | 字段 | 值 |
   |------|-----|
   | 服务名称 | `recipe-api` |
   | 部署方式 | **代码包上传** |
   | 代码包 | 打包 `e:\GO\recipe\wechat\` 目录为 `.zip` 上传 |

3. **先别点部署**，继续看下一步配置环境变量

### 1.3 配置环境变量

在创建服务时（或部署后在「服务设置 → 环境变量」），添加以下变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `PORT` | 服务端口 | `80` |
| `DB_HOST` | MySQL 内网地址 | `10.0.x.x`（从云开发 MySQL 面板复制） |
| `DB_PORT` | MySQL 端口 | `3306` |
| `DB_USER` | 数据库用户名 | `root` |
| `DB_PASS` | 数据库密码 | 开通 MySQL 时设置的密码 |
| `DB_NAME` | 数据库名 | `recipes` |

> ⚠️ `DB_NAME` 指定的库需要提前在 MySQL 中创建：
> ```sql
> CREATE DATABASE IF NOT EXISTS recipes DEFAULT CHARACTER SET utf8mb4;
> ```
> 也可以先在云开发 MySQL 面板的 WebSQL 中执行这条 SQL。

配置完成后点击 **「部署」**，等待 3-5 分钟构建完成。

### 1.4 验证后端是否正常

部署成功后：
1. 云托管面板 → 点击 `recipe-api` 服务 → 查看 **「日志」**
2. 应该看到类似输出：
   ```

4. 浏览器访问 `https://你的域名/api/categories`，应该返回 JSON 分类列表

## ⚠️ 上线前必须处理的问题

### 🔴 高优先级

| 问题 | 现状 | 修复方案 |
|------|------|---------|
| **用户 ID 写死** | `userId: 'demo_user_001'`，所有用户共享数据 | 在 `app.js` 中调用 `wx.login()` 获取 openid 作为 userId |
| **MySQL 密码安全** | 环境变量明文存储 | 云托管支持「Secret 环境变量」，把 `DB_PASS` 设为加密变量 |

### 🟡 中优先级

| 问题 | 现状 | 修复方案 |
|------|------|---------|
| **API 无鉴权** | 所有接口无身份校验 | 添加 JWT 中间件或微信登录态校验 |
| **外链封面图** | `AssignImages` 使用 Unsplash 外链 | 改成上传到微信云存储 `wx.cloud.uploadFile` |

### 🟢 低优先级 / 未来优化

| 优化项 | 说明 |
|--------|------|
| 图片上传迁移云存储 | 目前上传到容器本地 `uploads/`，容器重启会丢失 |
| 增加管理后台 | 方便运营维护菜谱、分类数据 |
| 接入微信云函数 | 把 seed 数据初始化等一次性任务改为云函数 |
